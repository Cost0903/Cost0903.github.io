# CI/CD 持續整合與部署

## 核心概念

### CI (Continuous Integration) - 持續整合
* **定義**：開發者將程式碼合併回主分支時，系統自動進行建置 (Build) 與測試 (Test)
* **目的**：及早發現錯誤，確保新程式碼不會破壞現有功能
* **在這個網站專案中**：
    當我們執行 `uv sync` 和 `uv run mkdocs build` 時，如果設定檔寫錯導致編譯失敗，CI 流程就會報錯通知我們

### CD (Continuous Deployment) - 持續部署
* **定義**：透過 CI 檢查後，自動將程式碼發佈到正式環境 (Production)
* **目的**：實現快速交付，減少人工部署的繁瑣與風險
* **在這個網站專案中**：
    GitHub Actions 自動執行 `mkdocs gh-deploy`，將編譯好的靜態網頁推送到 `gh-deploy` 分支，即時更新網站

## 為什麼後端工程師需要懂
1. **減少 Manual Toil (重複勞動)**：把時間花在寫程式，而不是下指令部署
2. **標準化流程**：避免「我在這台電腦跑可以，那台就不行」的問題
